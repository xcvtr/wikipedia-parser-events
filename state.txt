Текущее состояние проекта:

1. Основная функциональность:
   - Скрипт парсит данные о катастрофах из Wikipedia
   - Обрабатывает как табличные данные, так и текстовые ссылки
   - Сохраняет результаты в CSV-файл и выводит их в консоль

2. Структура данных:
   - События содержат следующие поля:
     - Death toll (количество жертв)
     - Event (название события)
     - City (город)
     - Country (страна)
     - Date (дата)
     - Details (детали)
     - Event Type (основной тип)
     - Event Subtype (подтип)
     - URL (ссылка на источник)

3. Основные функции:
   - clean_disaster_type() - очищает тип катастрофы из URL
   - format_date() - стандартизирует формат дат
   - extract_city() - извлекает город из текста
   - determine_country() - определяет страну из текста и URL
   - create_short_event_name() - создает краткое название события
   - determine_event_type() - классифицирует тип и подтип катастрофы
   - parse_table() - парсит табличные данные
   - extract_links_from_text() - извлекает события из текстовых ссылок

4. Последние улучшения:
   - Добавлена поддержка определения страны из URL
   - Улучшена нормализация названий стран
   - Добавлена обработка длинных названий событий
   - Реализована поддержка различных форматов дат

5. Текущие ограничения:
   - Некоторые города и страны могут оставаться как "Unknown"
   - Не все даты могут быть корректно распознаны
   - Возможны дубликаты в данных

6. Формат вывода:
   - Консольный вывод с подробной информацией о каждом событии
   - CSV-файл с структурированными данными
   - Статистика по типам катастроф, подтипам и странам

7. Зависимости:
   - requests (для HTTP-запросов)
   - beautifulsoup4 (для парсинга HTML)
   - pandas (для обработки данных)
